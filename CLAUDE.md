# HelloEmpathy Website Project

## Purpose
This project is for rebuilding the Hello Empathy website.

## Project background
- The earlier Hello Empathy site was a simple static website hosted on GitHub Pages with a custom domain.
- Favor a structure that is easy to understand, edit, and maintain.
- Preserve simple deployment unless we intentionally choose a more advanced stack.

## Goals
- Create a clean, modern, warm, professional site.
- Prioritize clarity, trust, and ease of navigation.
- Keep the codebase simple and maintainable.
- Make the site responsive for mobile and desktop.

## Design preferences
- Calm, clean, human-centered feel.
- Strong visual hierarchy.
- Good spacing and readable typography.
- Avoid clutter.
- Friendly, compassionate, trustworthy tone.
- Professional without feeling cold or corporate.

## Coding preferences
- Prefer clear, maintainable code over clever code.
- Keep components or sections modular.
- Comment only when helpful.
- If introducing a new dependency, explain why first.
- Prefer simple architecture unless there is a clear benefit to adding complexity.

## Technical guardrails
- Prefer a simple, maintainable website architecture.
- Keep deployment compatibility in mind and avoid unnecessary complexity.
- Avoid backend-dependent features unless approved first.
- If recommending a framework, explain why it is worth the added complexity.
- Prefer performance, readability, and ease of maintenance over flashy implementation.

## Working style
- Explain the plan before making major edits.
- Prefer small, reviewable changes.
- Keep file structure organized and easy to understand.
- Do not add unnecessary complexity.
- Before broad redesigns, summarize what will change.

## Website content priorities
- Make it easy for a visitor to understand who the practice serves.
- Make it easy to understand services and next steps.
- Build trust through tone, clarity, and clean presentation.
- Keep copy readable and emotionally grounded.

## Brand and visual direction
- Use the Hello Empathy logo in `assets/brand/logo.jpg` as the primary brand mark.
- Preserve the existing brand feel: calm, warm, professional, and trustworthy.
- Use the current green-and-gold color direction as the starting palette for the site.
- Avoid introducing colors that clash with the logo.
- Keep the visual style clean, spacious, and human-centered.
- Do not stretch, distort, or restyle the logo without asking first.
- Read `assets/brand/brandnotes.md` when making visual or branding decisions.

## Brand asset workflow
- Keep brand files in `assets/brand/`.
- Treat `assets/brand/logo.jpg` as the primary logo source unless another file is explicitly designated.
- If additional brand assets are added, prefer reusing them rather than recreating them.
- Ask before changing logo treatment, brand colors, or typography direction in a major way.

## Likely page structure
- Home
- About
- Services
- Resources
- Contact

## Workflow
- When starting a task, first identify the files likely involved.
- Suggest the cleanest approach before editing.
- For UI changes, describe what will change before making edits.
- For content changes, preserve warmth and clarity.
- Review `CLAUDE.md` and any relevant files in `assets/brand/` before proposing major design updates.
